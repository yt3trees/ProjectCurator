  LLM機能のアイデア

  A. 既存パターンの自然な拡張(実装しやすい)

  1. Decision Log 自動起票
  - Editor で current_focus.md を編集中に「Dec Log」ボタンの隣に AI ボタンを追加
  - current_focus.md + 直近の変更差分から「暗黙の意思決定」を検出し、decision_log
  のテンプレート(TEMPLATE.md)に沿った構造化エントリを提案
  - 既に .claude/skills/context-decision-log で定義されている概念をアプリ内に持ってくる形

  2. Project Summary 自動生成/更新
  - current_focus.md、decision_log、tensions.md、asana-tasks.md を入力にして project_summary.md を再生成
  - FocusUpdateService と同じ diff プレビュー + 承認フローを再利用できる
  - プロジェクト作成直後の初期サマリ生成にも使える

  3. Tensions.md 自動更新
  - Asanaタスクの期限超過、focus の長期未更新、decision_log の未解決項目から「緊張点」を検出
  - 既存ファイルにマージする形で提案

  ---
  B. クロスプロジェクト視点(中程度の挑戦)

  4. Smart Daily Standup
  - 現在の StandupGeneratorService はテンプレート的な生成だが、LLM に渡して自然言語の standup を生成
  - 複数プロジェクトの focus_history diff + 完了タスク + Today Queue を横断的に要約
  - 「昨日何をして、今日何をするか」を人間が読みやすい文章に

  5. Weekly Digest / ふりかえりレポート
  - 1週間分の focus_history、decision_log、完了タスクを集約
  - プロジェクトごとの進捗サマリ + 全体のハイライト/リスクを生成
  - Obsidian に週次ノートとして保存

  6. Dashboard AI サマリ
  - Dashboard 上部に「今日の全体状況」を1-2行で表示
  - 全プロジェクトの FocusAge、未コミット数、Overdue タスク数などを入力に、注意すべきプロジェクトを自然言語でハイライト
  - 例: 「ProjectX の focus が5日未更新、ProjectY に期限超過タスク3件」

  ---
  C. タスク/優先度の知的支援

  7. Today Queue 優先順位の提案
  - Today Queue のタスク一覧 + 各プロジェクトの current_focus.md を渡して、「今日これからやる順番」を提案
  - 依存関係、期限、プロジェクトの緊急度を考慮
  - Dashboard の Today Queue ヘッダーに「AI Sort」ボタン

  8. タスク分解アシスタント
  - Today Queue の大きなタスクを選択 → LLM がサブタスク案を生成
  - 生成結果を Asana API 経由で直接サブタスクとして登録(Asana書き込みは挑戦的だが実用性高い)

  ---
  D. 挑戦的でおもしろいもの

  9. Focus Drift 検出
  - focus_history の時系列変化を分析し、「目標からのドリフト」を検出
  - 例: 「2週間前は認証基盤の設計がメインだったが、最近はバグ修正ばかりになっている」
  - Timeline ページに AI インサイトとして表示

  10. プロジェクト間コンテキストスイッチ支援
  - Dashboard でプロジェクトを切り替える際、「前回どこまでやったか」「今の状態」「次にやるべきこと」を LLM
  が要約してポップアップ表示
  - current_focus.md + 最新の decision_log + git の未コミット変更を入力に

  11. 自然言語コマンド (Command Palette 拡張)
  - Command Palette に自然言語入力モードを追加
  - 「GenAi プロジェクトの先週の決定事項を見せて」→ decision_log をフィルタして要約
  - 「期限が近いタスクだけ表示して」→ Today Queue をフィルタ
  - アプリ内の操作を自然言語で実行できるエージェント的な機能

  12. Git コミットメッセージからの自動 Focus 更新
  - Git Repos ページで検出した未コミット変更 or 最近のコミットログを読み取り
  - 「コードレベルで何をしたか」を current_focus.md に反映する提案
  - Asana タスクだけでなく、実際の開発作業からも focus を更新
